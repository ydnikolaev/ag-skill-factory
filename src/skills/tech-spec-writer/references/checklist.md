# Tech Spec Checklist

Use this checklist before handing off a specification to developers.

## Content Completeness

- [ ] **Behavior** described in natural language
- [ ] **Edge cases** table with all error scenarios
- [ ] **Pseudocode** covers main logic flow
- [ ] **API examples** include request/response for all statuses
- [ ] **TDD Hints** provided (what to mock, test first, assert)

## Quality Checks

- [ ] No actual code (only pseudocode)
- [ ] User can understand without reading code
- [ ] All inputs/outputs documented
- [ ] Dependencies listed

## Handoff Ready

- [ ] Spec saved to `docs/specs/<feature>-tech-spec.md`
- [ ] User approved via `notify_user`
- [ ] `docs/ARTIFACT_REGISTRY.md` updated
